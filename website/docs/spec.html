Specification

The contract between intent, tools, and agents.

The current GoalOS spec defines a structured intent graph, the permissions model around it, and the transport surfaces that let external clients read and write context safely.

Surface area

  • TypeScript core - source of truth for graph operations and validation.
  • Python SDK - Pydantic models plus async-friendly client access.
  • MCP server - tool surface for Claude Desktop and other clients.
  • CLI - local inspection, export, and workflow commands.

Core entities

EntityPurpose
GoalA unit of intent with status, priority, progress, and hierarchy.
PriorityHow the graph ranks competing work and time horizons.
PermissionWhat an agent can read, write, complete, or reprioritize.
EventHistory for changes, merges, and lifecycle transitions.

Serialization

The repo includes JSON-LD and schema-driven representations so the graph can move across tools without losing meaning.

{ "type": "IntentGraph", "version": "0.1.0", "goals": [], "permissions": [], "events": [] }

Validation rules

Structure first

Goals need stable identifiers and clean parent-child relationships.

Permissions explicit

Agent access should be granted intentionally, not inferred.

Exportable data

Consumers should be able to inspect and move the graph without special runtime state.

Transport neutral

CLI, SDKs, and MCP all expose the same intent model.
