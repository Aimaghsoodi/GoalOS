Manifesto

AI tools should know what you are trying to do.

GoalOS exists because memory, chat logs, and prompt history are not enough. Tools need a shared, machine-readable model of intent so they can cooperate instead of guessing.

Your tools are powerful. They just do not know what you want. GoalOS makes intent explicit, portable, and enforceable.

What this means

One graph Goals, priorities, dependencies, and permissions in one place.
Many clients Claude Desktop, LangChain, CrewAI, CLI, and Python all read the same intent.
Less repetition Define context once instead of re-explaining it every session.
Safer automation Permissions keep agents scoped to the exact goals they should touch.

Principles

Intent over inference. Shared state over repeated prompting. Structured data over fragile prose. Permissions before automation. Interoperability before lock-in.

What it is not

It is not a chatbot memory dump, a black-box agent platform, or another disconnected note system. GoalOS is a protocol and runtime contract for goals.

For builders

Use GoalOS when you want agents, scripts, and UIs to act on the same source of truth. The API surface stays small, typed, and predictable.

For users

Use GoalOS when you want your AI stack to remember what matters and stop optimising locally against stale assumptions.
