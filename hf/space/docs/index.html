Overview

What GoalOS actually is

GoalOS is a protocol and package surface for representing human intent in a structured graph that AI systems can reason over. It models goals, dependencies, permissions, status, priorities, and time horizons so multiple tools stop acting like isolated copilots and start coordinating against the same operational picture.

Important: GoalOS is not just memory and not just task tracking. It is the intent layer that sits above both.

Quickstart

Install the surface that matches your stack

TypeScript core 
npm install @goalos/core
Python SDK 
pip install goalos
CLI 
npm install -g goalos
MCP server 
npm install -g @goalos/mcp-server
TypeScript example 
import { IntentGraphClass } from '@goalos/core';

const graph = IntentGraphClass.create('founder@abteex.com', 'Launch Board');

const launch = graph.addGoal({
  title: 'Launch GoalOS public demo',
  status: 'active',
  priority: { level: 'critical' },
  domain: 'product',
  successCriteria: ['Space is live', 'Docs are public'],
});

graph.addGoal({
  title: 'Ship Hugging Face cinematic demo',
  parentId: launch.id,
  status: 'active',
  priority: { level: 'high' },
  tags: ['launch', 'demo'],
});
Python example 
from goalos import IntentGraphManager, Priority

graph = IntentGraphManager.create(owner="abteex", name="Personal Chief of Staff")

graph.add_goal(
    title="Protect deep work block",
    priority=Priority(level="high", score=80),
    domain="personal_ops",
)

Surfaces

Package surfaces

@goalos/core

The canonical TypeScript engine for goals, graphs, priority, validation, merge logic, permissions, and queries.

goalos CLI

Terminal-first graph creation, inspection, status review, prioritization, validation, import/export, and serving.

@goalos/mcp-server

Connect GoalOS to Claude Desktop and other MCP clients so they can read and update intent-aware context.

goalos Python SDK

Python-native graph modeling and automation workflows with strong typing and API-friendly data structures.

Model

Core graph semantics

  • Goal: a unit of intent with status, domain, priority, time horizon, and success criteria.
  • Hierarchy: parent-child relationships for strategic goals and execution work.
  • Dependency: explicit blockers, enablers, and related goals across teams or tools.
  • Permissions: scoped capabilities so agents do not get global write access by default.
  • Priority engine: dynamic urgency based on deadlines, blockers, and current execution state.

Publish

How this Hugging Face Space is deployed

This repo includes a self-contained Static Space bundle in hf/space/ plus helper scripts for verification and publishing.

Local commands 
node scripts/verify-hf-space.mjs
node scripts/publish-hf-space.mjs

The publish helper creates or reuses AbteeXAILab/GoalOS as a public Static Space and uploads the contents of hf/space/.
